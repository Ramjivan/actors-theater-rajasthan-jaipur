# Deployment & Infrastructure Documentation

This document outlines the technical details for deploying and maintaining the Actors Theatre Rajasthan (ATR) website.

## Tech Stack
- **Framework:** Astro (SSR mode)
- **Adapter:** `@astrojs/cloudflare`
- **Hosting:** Cloudflare Workers
- **CMS:** Decap CMS (formerly Netlify CMS)
- **CI/CD:** GitHub Actions

---

## 1. CI/CD & Auto-Deployment
The website automatically builds and deploys to Cloudflare Workers every time code is pushed to the `main` branch on GitHub.

This is handled by the GitHub Action located at `.github/workflows/deploy.yml`.

### Required GitHub Secrets
For the GitHub Action to successfully push the code to Cloudflare, the following repository secrets must be configured in **GitHub -> Settings -> Secrets and variables -> Actions**:

1. `CLOUDFLARE_ACCOUNT_ID`: Your Cloudflare Account ID (found on the right sidebar of the Cloudflare dashboard).
2. `CLOUDFLARE_API_TOKEN`: A custom Cloudflare API token with the correct permissions (see below).

### Cloudflare API Token Permissions
When generating the `CLOUDFLARE_API_TOKEN` in the Cloudflare Dashboard (My Profile -> API Tokens -> Create Custom Token), the token **must** have the following minimum permissions:

- **Account** | **Workers Scripts** | **Edit** *(Required to upload the code)*

*Recommended additional permissions if Wrangler needs to modify domain routes or read account details:*
- **Account** | **Account Settings** | **Read**
- **User** | **User Details** | **Read**
- **Zone** | **Workers Routes** | **Edit**

## 2. Decap CMS & Authentication
The website uses **Decap CMS** for content management, allowing non-technical users to edit site content. Decap CMS relies on GitHub for authentication and data storage.

### OAuth Proxy Setup
Because the site is not hosted on Netlify, Decap CMS requires an external OAuth Proxy to authenticate users with GitHub securely.

1. **GitHub OAuth App:** An OAuth App must be created in GitHub (Settings -> Developer settings -> OAuth Apps).
2. **Environment Variables:** The OAuth Client ID and Client Secret generated by GitHub must be securely stored in your Cloudflare dashboard as environment variables for the project. These should **never** be committed to the code repository.

**Required Environment Variables in Cloudflare:**
- `OAUTH_GITHUB_CLIENT_ID`
- `OAUTH_GITHUB_CLIENT_SECRET`

*Note: Depending on the specific OAuth proxy implementation being used, these variable names may slightly differ. Ensure they match your specific proxy setup.*

## 3. Local Development
To run the project locally and view changes before pushing:

1. Install dependencies:
   ```bash
   npm install
   ```
2. Start the local development server:
   ```bash
   npm run dev
   ```
3. To test a production build locally:
   ```bash
   npm run build
   npm start
   ```
